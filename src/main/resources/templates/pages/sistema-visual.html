Colores

Paleta con códigos HEX, variaciones y usos recomendados.

  • Primario (brand/emerald): CTAs, links e iconos principales.
  • Accento (teal-600): destacar acciones secundarias o badges.
  • Feedback: success (emerald-600), error (rose-600), warning (amber-500), info (sky-600).
  • Neutros (slate): títulos y textos.
  • Fondo (white): tarjetas y superficies.
  • Overlays (brand/opacity): fondos con brand/40 en hero.
  • Ring: bordes sutiles (emerald-900/10..20, white/15 en hero).

Cuándo usar
  • Brand: CTAs, links primarios, indicadores positivos
  • Accento (teal-600): llamadas secundarias, badges
  • Feedback: success/error/warning/info según estado
  • Slate: tipografía (900 títulos, 700 párrafos)
Variaciones
  • Hover: emerald-700
  • Deshabilitado: colores a 40–60% de opacidad
  • Rings: emerald-900/10..20
Accesibilidad
  • Contraste mínimo 4.5:1 para texto
  • No usar sólo color para estado

Cuándo NO usar

Ejemplos de anti‑patrones a evitar.

Botones
  • NO usar más de un botón primario por bloque.
  • NO usar primario con baja opacidad como estado normal.
  • NO usar múltiples colores de énfasis simultáneos sin jerarquía.
Tipografía
  • NO usar todo en MAYÚSCULAS en párrafos largos.
  • NO usar slate-300 para texto de contenido.
  • NO mezclar más de 2 tamaños de título en la misma tarjeta.
Colores
  • NO combinar brand con otros énfasis sin jerarquía (confunden prioridad).
  • NO usar brand sobre fondos oscuros sin verificar contraste.
  • NO usar saturaciones extremas en fondos de lectura.
Sombras
  • NO usar más de un nivel de sombra simultáneo.
  • NO usar sombras fuertes en elementos pequeños (badges).
  • NO usar sombras para simular bordes; usar ring.
